# Introduccion 

Un programa necesita datos en los cuales operar. A lo largo de estas lecciones se han estado escribiendo todos los valores de datos en el programa mismo, tanto como en variables como en literales. Si esta fuera la única forma de introducir datos, se tendría que rescribir el programa cada vez que se quisiera aplicarlo a un conjunto distinto de valores. Para abordar este tema con mas profundidad, nos meteremos de lleno con la *entrada y salida* 

La entrada y salida de datos es una parte fundamental de cualquier programa, ya que permite que exista comunicación entre el usuario y la computadora. A través de estos procesos, un programa puede recibir información, procesarla y después mostrar resultados de forma comprensible.

En C++, la entrada y salida se manejan principalmente mediante *flujos de datos*, que son herramientas que facilitan el intercambio de información entre el programa y dispositivos externos, como la consola. Para ello, normalmente se utilizan objetos como `cin` para la entrada y `cout` para la salida. Gracias a ellos, el programador puede pedir datos al usuario, almacenar valores en variables y luego presentar mensajes o resultados en pantalla. Esto hace que los programas sean más útiles, dinámicos e interactivos.

La importancia de este tema radica en que casi todos los programas necesitan, en algún momento, leer información o mostrar resultados. Por ejemplo, una aplicación puede solicitar el nombre de una persona, capturar su edad y después mostrar un mensaje personalizado. También puede pedir números para realizar cálculos y mostrar el resultado final. De esta manera, la entrada y salida no solo permiten que el programa funcione, sino que también hacen posible que el usuario participe activamente en su ejecución.

A lo largo de estas lecciones entenderás de mejor manera como interactua con la entrada y la salida, lo cual te ayudará a cómo interactuar con un programa de mejor manera, lo que por consiguinete, te servirá como base para realizar actividades más avanzadas y construir programas más complejos.
